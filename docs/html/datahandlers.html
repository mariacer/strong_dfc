General Notes¶

Datahandlers for different datasets are specified in this module. They should all inherit from the common abstract class datahandlers.dataset.

API¶

Implementation of a general Dataset¶

Extremely simple implementation of a dataset wrapper.

class datahandlers.dataset.DatasetWrapper(trainset, testset, valset=None, name=None, in_size=None, out_size=None)[source]¶

Bases: object

Implementation of a simple dataset wrapper.

Besides containing the train, test and validation sets for a given task, some information about the dataset is also contained, like input and output size, and whether the targets exist.

Parameters

  • trainset (torch.utils.data.Dataloader) – The train set.

  • testset (torch.utils.data.Dataloader) – The test set.

  • valset (torch.utils.data.Dataloader) – The (optional) validation set.

  • name (str) – The name of the dataset.

  • in_size (str) – The input size.

  • out_size (str) – The output size.

MNIST Dataset¶

Implementation of dataloaders for the MNIST dataset.

class datahandlers.mnist_data.MNISTData(root, device, double_precision=True, train=True, target_class_value=1, **kwargs)[source]¶

Bases: torchvision.datasets.mnist.MNIST

An instance of the class represents the MNIST dataset.

Parameters

  • root (str) – Root directory of dataset

  • device – The cuda device where to place the dataset.

  • double_precision (boolean) – Whether precision of 64 floats is used.

  • train (boolean) – Whether to create the dataset from training data.

  • target_class_value (float) – The value of the correct class for the targets. If set smaller to one, can be used to set soft targets. Note that unless a value different than 1 is used, the labels are simply the integer values, not one-hot encodings.

Teacher-based Dataset¶

Implementation of dataloaders for the a teacher network-based dataset.

class datahandlers.student_teacher_data.RegressionDataset(device, n_in=5, n_out=5, n_hidden=[100, 100], num_data=3000, activation='tanh', double_precision=True, random_seed=42)[source]¶

Bases: torch.utils.data.dataset.Dataset

A teacher network-based regression dataset.

In this setting, a teacher network is generated, and the dataset is obtained by feeding random inputs to this teacher network.

Parameters

  • device – The cuda device where to place the dataset.

  • n_in (int) – The dimensionality of the inputs.

  • n_out (int) – The dimensionality of the outputs.

  • n_hidden (list) – The dimensionality of the hidden layers.

  • num_data (int) – The number of datapoints to generate.

  • activation (str) – The activation function to be used.

  • double_precision (boolean) – Whether precision of 64 floats is used.

  • random_seed (int) – The random seed for data generation.
